## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Health check with DB status |
| GET | `/api/storefront` | Store info + featured products |
| GET | `/api/products` | All products |
| GET | `/api/products/<id>` | Single product |
| POST | `/api/orders` | Create an order |
| GET | `/api/orders/<id>` | Get order by ID |

## Deploy to Render

1. Push this project to a GitHub (or GitLab) repository.
2. Go to [render.com](https://render.com) в†’ **New в†’ Web Service**.
3. Connect your repository.
4. Render auto-detects `render.yaml` вЂ” click **Apply**.
5. Set the required environment variables in the Render dashboard:

| Variable | Description |
|----------|-------------|
| `BAKEWISE_DB_HOST` | Your MySQL host (e.g. from PlanetScale, Railway, or your own server) |
| `BAKEWISE_DB_PORT` | MySQL port (default `3306`) |
| `BAKEWISE_DB_USER` | MySQL username |
| `BAKEWISE_DB_PASSWORD` | MySQL password |
| `BAKEWISE_DB_NAME` | Database name (e.g. `bakewise`) |
| `SECRET_KEY` | Auto-generated by render.yaml, or set your own |

> **Tip:** Use a managed MySQL service like [PlanetScale](https://planetscale.com), [Railway](https://railway.app), or [Aiven](https://aiven.io) for a cloud database Render can reach.

6. Click **Deploy**. Your site will be live at `https://<your-app>.onrender.com`.

## Run Locally

1. Copy the example env file and fill in your database details:
   ```bash
   cp .env.example .env
   # edit .env with your MySQL credentials
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Start the server:
   ```bash
   python run.py
   ```

4. Open: `http://localhost:5000`

## Run with Docker

```bash
docker build -t bakewise-web .
docker run -p 5000:5000 --env-file .env bakewise-web
```

## Notes

- Product images are generated as SVG placeholders. Add an `image_url` column to your products table and update the API to serve real photos.
- Orders are saved as `service_mode = Online Orders` so BakeWise staff can process them in the existing workflow.
- The cart is stored in `localStorage` вЂ” no login required.
